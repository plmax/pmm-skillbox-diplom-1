# Информационно-техническая инфраструктура организации с пользователями работающими удаленно

## Описание проекта

Создание инфрастуктуры для централизованного управления учётными записсями пользователей.
Доступ к внутренним ресурсам сети компании будет только у сотрудников через VPN.
Будет проводится мониторинг инфрастурктуры для предупреждения неисправностей.
Будет проводится резервное копирование критических данных инфрастурктуры.
Будут задокументированы настройки всех компонентов инфрастурктуры.
Будут созданы необходимые руководства для пользователей по пользованию инфрастурктурой.

## Требования к инфраструктуре

Все компоненты инфраструктуры создаются на базе Ubuntu 24.04 LTS на виртуальных серверах хостера или физической инфрастурктуры организации.
При развертывании виртуальных серверов из шаблона предусмотреть настройку доступа по SSH.

## Как пользоваться и развернуть ваш проект

Для развёртывания инфраструктуры необходимо склонировать данный проект на хост инфраструктуры, выбрать роль согласно [списку ролей (серверов)](#состав-серверов-ролей-проекта), использовать скрипт развертывания `init-role.sh` из каталога выбранной роли.

### Состав серверов (ролей) проекта

1. ca-srv - Сервер удостоверяющий центр инфраструктуры для выдачи сертификатов
2. vpn-srv - Сервер для создания защищенного соединения между пользователем и инфраструктурой
3. mon-srv - Сервер для развёртывания системы мониторинга
4. backup-srv - Сервер хранения резервных копий критических данных

### Общие настройки

Для каждого компонента (роли) инфраструктуры созданы [общие элементы настройки](0.Общие_настройки)

1. `init-vm.sh` - скрипт для обновления системы и установки общих пакетов (ufw, fail2ban)

   - ufw - пакет управления файрволлом
   - fail2ban - пакет защиты хоста от взлома методом подбора пароля и    других видов несанкционированного доступа

2. ufw-setup-1.0.deb - установка общих настроек для пакета ufw

3. fail2ban-config.deb - установка общих настроек для пакета fail2ban
